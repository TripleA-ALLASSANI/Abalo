Clients(app mob, navigateur, tv,...)-FAI-Internet-Serveurs Les serveurs connectés directement à Internet(les pages Web (contenu) sont stockées sur le disque dur de ces serveurs(serveurs Web, les serveurs de messagerie et les serveurs virtuels).)  
Les serveurs sont des programmes ou des systèmes qui s'exécutent sur un ordinateur spécial (avec un matériel spécial comme un équipement réseau et un accès réseau dédié).
Important protocoles: HTTP && TCP/IP && DHCP
	-Http: définit la façon dont les messages sont formatés et transmis. Il détermine également les actions que les serveurs Web et les 		navigateurs doivent effectuer en réponse à diverses commande.
	-IP fournit un ensemble standard de règles pour l'envoi et la réception de données sur Internet.
		Pour être reconnus par d'autres appareils, les hôtes connectés à Internet doivent avoir une adresse IP .
		IP est chargé de s'assurer que les paquets sont envoyés à la bonne destination.
		Le protocole Internet fournit également des instructions de base pour le transfert de paquets entre les appareils.
	-TCP est un protocole fondamental de la suite de protocoles Internet.
		TCP est le composant qui collecte et réassemble les paquets de données.
	-Le TCP/IP est catégorisé comme un protocole de "couche de transport" car il crée et maintient des connexions entre les hôtes.
	-DHCP est un protocole qui attribue automatiquement une adresse IP unique à chaque appareil qui se connecte à un réseau.

API:est un ensemble de code de programmation(logiciel) qui permet la transmission de données entre un produit logiciel(application) et un autre
	-Importance: peuvent simplifier et accélérer le développement de logiciels.
	-Les API Web sont des API accessibles via le protocole HTTP.
	-REST (representational state transfer) est une architecture d'API Web très populaire.
	-L'API REST est une API qui doit avoir certaines contraintes ou principes architecturaux, notamment : 
		Architecture client-serveur*Apatride*Capacité de mise en cache*Système en couches.
	-L'architecture REST utilise les instructions d'origine du protocole HTTP.

		Règle n°1 : l'URI doit fonctionner comme un identifiant de ressource.
		Règle n°2 : les verbes HTTP doivent fonctionner comme les identifiants des opérations.
		Règle n°3 : les réponses HTTP doivent fonctionner comme des représentations de ressources.
		Règle n°4 : les liens sont comme une relation entre des ressources.
		Règle n°5 : L'existence d'un paramètre comme un jeton d'authentification

Un NoSQL“not only SQL,”. Les bases de données NoSQL(qui regroupe des systèmes de base de données qui ne sont pas relationnels) ont été conçues par des sociétés technologiques telles que Google, Amazon, Yahoo et Facebook pour fournir des moyens plus efficaces de stocker du contenu ou de traiter des données pour d'énormes sites Web.

Diffrence entre BD SQL et BD NoSQL
	Les bases de données SQL sont basées sur des tables, tandis que les bases de données NoSQL sont des magasins de documents, 			de valeurs-clés, de graphiques ou de colonnes étendues.
	Les bases de données SQL sont relationnelles, NoSQL ne sont pas relationnelles.

Sélecteur CSS( Class selector): Pour indiquer les éléments que nous essayons de modifier
	comment attacher un style choisi à l'élément approprié dans le but de changer le style d'un élément particulier de notre page HTML.
Types de sélecteur :
	https://developer.mozilla.org/fr/docs/Learn/CSS/Building_blocks/Selectors/Type_Class_and_ID_Selectors
	Les trois types de sélecteurs de base sont :
		Sélecteurs de type,Sélecteurs de classe,Sélecteurs d' identifiant.
		Sélecteur de type : est utilisé lorsque l'on veut appliquer la même règle sur chaque élément de la page( e.g tout les h1).
		Sélecteurs de classe : commence par un point . et sélectionne tout élément du document auquel cette classe est appliquée.
		Sélecteur d'identifiant : la sélection par Id.Les sélecteurs d'ID sont uniques, nous ne pouvons donc les appliquer qu'au 			contenu d'un élément. Pour référencer un ID, nous faisons précéder le nom de l'ID d'un dièse (#)
	Autres types de sélecteurs:
		Sélecteurs multiples : SS nous permet de  sélectionner plusieurs éléments, séparer les sélecteurs par des virgules afin qu'ils 			  partagent la meme propriéteé, ainsi respecter le principe « DRY » de la programmation(i.e: éviter les répetitions inutiles).
		Combinateur de descendants(est généralement représenté par un seul caractère espace) : 
			Il combine deux sélecteurs de manière à ce que les éléments correspondant au deuxième sélecteur 				soient sélectionnés s'ils ont un élément ancêtre (parent, parent du parent, parent du parent du parent, etc.) 				correspondant au premier sélecteur. Les sélecteurs qui utilisent un combinateur descendant sont appelés sélecteurs 				descendants.
		Combinateur enfant(>)  : Pour sélectionner uniquement les enfants d'un certain élément parent 
			Les éléments descendants plus bas dans la hiérarchie ne correspondent pas.
			Par exemple, pour sélectionner uniquement les <p>éléments qui sont des enfants directs d'<article>éléments.
		Combinateur de frères et sœurs adjacents(+) : Par exemple, pour sélectionner tous les <p>éléments qui viennent juste après les 				<h1>éléments :
		Combinateur général de frères et sœurs (~): Pour sélectionner tous les <p>éléments qui viennent n'importe où après les 				<h1>éléments.

Modèle de boîte CSS(Basic Box Model): Chaque élément de la conception Web est un rectangle en forme de boîte (même s'il s'agit d'un cercle).
Le modèle de boîte CSS est défini par ces quatre couches : CRBM(Content Padding Border Margin)

	Content : le contenu de la boîte, où apparaissent le texte et les images
	Padding : La padding propriété en CSS définit la partie la plus interne du modèle de boîte , créant un espace autour du contenu 			  d'un élément, à l'intérieur de toutes les marges et/ou bordures définies.
		Les valeurs de remplissage sont définies à l'aide de longueurs ou de pourcentages et ne peuvent pas accepter de valeurs 		  négatives. La valeur initiale, ou par défaut, de toutes les propriétés de remplissage est 0.
		Padding : la propriété padding pour créer un espacement entre la zone de contenu et la bordure d'un élément.
			https://css-tricks.com/almanac/properties/p/padding/
	Border : permet de spécifier à quoi doit ressembler la bordure de la boîte représentant un élément.
		https://www.tutorialspoint.com/css/css_borders.htm
		Border-radius: arrondit les coins de la bordure extérieure d'un élément.
			https://developer.mozilla.org/en-US/docs/Web/CSS/border-radius
	Margin : efface une zone en dehors de (border). La marge est transparente.
		Fondamentalement, les marges définissent la partie la plus externe du modèle de boîte, créant un espace autour d'un élément, 				en dehors de toute bordure définie.
		Les marges sont définies à l'aide de longueurs, de pourcentages ou du mot-clé auto et peuvent avoir des valeurs négatives.
			https://css-tricks.com/almanac/properties/m/margin/

	JavaScript est un langage de programmation orientée objet (OOP). 
		Un langage de programmation peut être appelé orienté objet s'il fournit quatre capacités de base aux développeurs: 
			Encapsulation - la capacité de stocker des informations connexes, qu'il s'agisse de données ou de méthodes, ensemble 					dans un objet.

//Exécuter une séquence d'instructions
			Il existe trois blocs  de base (constructions) à utiliser lors de la conception d'un algorithme : 					sequencing, selection, iteration - "séquençage, sélection, itération"
			1- Séquençage 
				le modèle de séquençage consiste à donner à l'ordinateur la bonne séquence d'instructions à suivre.
				NB: un ordinateur ne peut exécuter que deux catégories d'instructions :
					. Opérations arithmétiques : addition, multiplication, etc...
					. Opérations logiques : dans ce type d'instructions l'ordinateur peut exécuter deux autres 							sous-catégories qui sont :
						Booléennes : ET, OU, NON , etc ...
						Relationnel : =,>,<=, etc...
				//Instructions arithmétiques
					 l'opération arithmétique c'est une manipulation de constantes ou de variables.
				//Instructions logiques
					Ces instructions effectuent des opérations logiques sur les données, telles que la comparaison, le 							complément et, ou...
			2- sélection
				Consiste est un ensemble d'instructions qui sont exécutées (réalisées) les unes après les autres. 
				La sélection nous permet d'inclure plus d'un chemin dans un algorithme.
				la sélection est généralement représentée par les instructions "IF , THEN et ELSE" .
					"IF" représente la question
					"THEN" indique ce qu'il faut faire si la réponse à la question est vraie
					"ELSE" indique ce qu'il faut faire si la réponse à la question est fausse

			3- itération
				L'itération dans la programmation signifie répéter des étapes, ou des instructions, encore et encore.
				C'est ce qu'on appelle souvent une " boucle " .
				"L'itération permet de simplifier les algorithmes en déclarant que certaines étapes se répéteront jusqu'à 						indication contraire. 
				Cela rend la conception d'algorithmes plus rapide et plus simple car ils n'ont pas besoin d'inclure de 						nombreuses étapes inutiles.
				Les boucles :   "WHILE-DO"  / "REPEAT-UNTIL" permet au code d'être exécuté de manière répétée en fonction 									d'une condition booléenne donnée.
						"FOR-DO" permet d'écrire une boucle qui est exécutée un certain nombre de fois.
				E.g:
					-WHILE-DO       	WHILE(i<= 10)
					-REPEAT-UNTIL	 	UNTIL(i>10)

//La programmation procédurale
	De milliers de lignes de code peuvent être générer, face à la résolution d'un problème complexe.
	"Solution": 
		Idée de programmation procédurale :
		Divisez le problème complexe en sous-problèmes moins complexes et écrire chacun d'eux séparément dans notre environnement 				approprié.
	"Avantages":
		Clarté de l'algorithme
		Un algorithme plus lisible
		Facilité d'entretien
		Réutilisation de sous-algorithmes
	L'idée derrière la programmation procédurale est de diviser le complexe Problème en moins complexe sous-problèmes et écrir chacun 			d'eux séparément dans son propre environnement.
//sous-algorithmes
	2 types de sous-algorithmes :
		Procédure
		Fonction
	la communication entre deux sous-algorithmes se fera via des paramètres (variables). Chaque sous-algorithme peut avoir des paramètres 			d'entrée et peut donner des paramètres de sortie.
		E.g: Read(), Write(), push() sont des procédures prédéfinies (également appelées intégrées).
		Si le programmeur crée les siens, ils sont personnalisés ou définis par l'utilisateur.
	//Diff entre procédure et fonction
		Une procédure exécute 1 tâche et retourne pas d valeur, tandis q'une fonction produit des informations puis retourne 1 et 1 				seule valeur.
	//Description de la procédure :
		PROCEDURE procedure_name(List_parameters)
		VAR
		    set_declarations
		BEGIN
		    Procedure’s body (set_instructions)
		END

	//Description of Function:
		FUNCTION function_name(args) : return_type
		VAR
		    set_declarations
		BEGIN
		    set_instructions
		    RETURN value ;
		END
	NB: Ces ééléments ( Fonctions, procédures, paramètres, arguments, variables et constantes) peuvent recevoir un identifiant.
	//Paramètres
		Nous permettent de transmettre des informations ou des instructions dans des fonctions et des procédures.
		Les valeurs passées en paramètres sont appelées arguments.
		NB: Nous pouvons avoir une fonction/procédure sans paramètres
	//scope - Portée des variables
		La portée fait référence à l'endroit où une fonction, une procédure, une variable ou une constante peut être utilisée. Ils 				peuvent être locaux ou mondiaux.
		Une variable globale est déclarée de manière à la rendre disponible pour toutes les autres parties du programme, 
		 Ce qui signifie que nous pouvons l'appeler dans n'importe quelle fonction/procédure sans passer en paramètre. 
		Alors qu'une variable locale est déclarée d'une manière qui la rend disponible uniquement dans une fonction ou une procédure 				spécifique, 
		Ce sont les variables déclarées dans la partie VAR de la fonction, de la procédure ou de l'algorithme principal lui-même.
		Nous n'avons pas de mot clé global pour déclarer des variables globales. 
		Cependant, les variables déclarées en dehors des fonctions, des procédures et de l'algorithme principal ont une

Donc si on modifie le tableau en cours de route alors les boucles pourront être 					impactées. Il existe aussi plusieurs autres méthodes sympas pr itérer sur les tableaux (comme "some ou every")
			-l'utiliser lorsque nous n'utilisons pas le tableau renvoyé(ou for...of à la place.)
				"e.g" :
					const people = [{ name: 'Max' }, { name: 'Jack' }, { name: 'Marry' }]
					// JavaScript
					function showEachOne(name) {
					 for (let i = 0; i < people.length; i++) {
					   alert(people[i].name)
					 }
					}
					// ES6
					const showEachOne = name => people.forEach(person => alert(person.name));
		3. filter()
			-filter() "crée un nouveau tableau avec tous les éléments qui réussissent le test implémenté par la fonction fournie.
			-Si nous considérons un "tableau de produits où chaque produit est un objet"
			-En appliquant la méthode du filtre , nous aurons un nouveau tableau qui éliminera tous les produits dont le prix est 					supérieur à 10 dollars.
				"e.g" :
					const products = [{name:"Milk",price:15}, {name:"Water", price:9}, {name:"Bread", price:5}];
					// JavaScript
					function filterProducts() {
					 let cheapProducts = [];
					 for (let i = 0; i < products.length; i++) {
					   if (products[i].price < 10) cheapProducts.push(products[i]);
					 }
					 return cheapProducts;
					}
					// ES6
					const filterProducts = () => products.filter(product => product.price < 10);
				"e.g" :
					const words = ['spray', 'limit', 'elite', 'exuberant', 'destruction', 'present'];
					const result = words.filter(word => word.length > 6);
					console.log(result);
					// expected output: Array ["exuberant", "destruction", "present"]
		4. map()
			ES6 fournit également une fonction "de transformation de tableau à l'aide de .map() .
			Utiliser lorsque nous utilisons le tableau renvoyé
			Supposons que nous voulions ajouter deux dollars au prix de chaque produit du tableau.
			"forEach" exécute la même instruction sur chaque élément d'un tableau donné et 
			la méthode ".map()" fait la même chose. 
			Cependant, "cela se fait sur une copie du tableau, en laissant les données d'origine inchangées.
				"e.g" :
					const array1 = [1, 4, 9, 16];
					// pass a function to map
					const map1 = array1.map(x => x * 2);
					console.log(map1);
					// expected output: Array [2, 8, 18, 32]
				"e.g" : 
					const products = [
					 { name: 'Milk', price: 15 },
					 { name: 'Water', price: 9 },
					 { name: 'Bread', price: 5 },
					]

					// JavaScript
					function changeProducts() {
					 for (let i = 0; i < products.length; i++) {
					   products[i].price += 2
					 }
					 return products
					}
					// ES6
					const changeProducts = () =>
					products.map(product => ({ ...product, price: product.price + 2 }));
					console.log(changeProducts());
					console.log(products);
		5. reduce()
			Le cas le plus simple à comprendre reduce() est peut-être de renvoyer la somme de tous les éléments d'un tableau.
			Elle parcourt le tableau élément par élément, ajoutant à chaque étape la valeur actuelle du tableau au résultat de 					l'étape précédente (ce résultat est la somme cumulée de toutes les étapes précédentes) — jusqu'à ce qu'il n'y 					ait plus d'éléments à ajouter.
			La fonction reduce() "exécute la fonction fournie pour chaque valeur d'un tableau de gauche à droite.
			La "valeur de retour(est une valeur unique)" est stockée dans"accumulator - un accumulateur.
				"e.g" :
					const data = [5, 10, 15, 20, 25]
					const res = data.reduce((total, currentValue) =>  total + currentValue)
					console.log(res)// 75
				"e.g" :
					const array1 = [1, 2, 3, 4];
					const reducer = (previousValue, currentValue) => previousValue + currentValue;
					// 1 + 2 + 3 + 4
					console.log(array1.reduce(reducer));
					// expected output: 10
					// 5 + 1 + 2 + 3 + 4
					console.log(array1.reduce(reducer, 5));
					// expected output: 15

			//Description de reduce()
				 - reduce() exécute la "fonction callback une fois pour chaque élément présent dans le tableau" et "ignore les 						éléments vides du tableau". 
				 - La fonction callback "utilise quatre arguments" :
					NB: accumulateur, valeurCourante sont les seuls obligatoires.
					. L'accumulateur (la valeur retournée par le précédent appel de la fonction callback), ou la valeur 							initiale s'il sagit du premier appel ;
					. la valeur de l'élément courant ;
					. l'index de l'élément courant ;
					. le tableau parcouru par la méthode.
					NB: si "valeurInitiale n'est pas fournie", reduce
						- exécutera la "fonction callback" à partir de l'indice 1 et
						- la première valeur du tableau (d'indice 0) sera utilisée pour valeurInitiale.
				e.g:
					[0, 1, 2, 3, 4].reduce(function(accumulateur, valeurCourante, index, array){
					  return accumulateur + valeurCourante;
					});
					//Code ES6 correspondant
					[0, 1, 2, 3, 4].reduce(
					  (accumulateur, valeurCourante) => accumulateur + valeurCourante;
					);
		

//Module Js  - (Named exports + Default exports) -  (import + export)
	https://medium.com/@etherealm/named-export-vs-default-export-in-es6-affb483a0910
	ES6 nous permet d'"importer un module et de l'utiliser dans d'autres fichiers.
	C'est pourquoi ES6 a introduit deux nouveaux mots-clés : "export et import" .
	Dans l'ensemble, la modularité "offre une plus grande facilité de gestion du développement logiciel.
	La modularité signifie que "nous pouvons diviser le code d'un programme en plusieurs morceaux réutilisables (généralement regroupés 			dans des fichiers). 
	En génie logiciel, la modularité fait référence à "la mesure dans laquelle une application logicielle/Web peut être divisée en modules 			plus petits.

	Les types JavaScript que nous pouvons exporter ou importer sont :

		String
		Number
		Function
		Objects - Objets
		Array
		Anything  - N'importe quoi ...
		
	//Type d'exportations
		Supposons que nous voulions utiliser un code créé dans un autre module, tout ce que nous avons à faire est "d'importer ce 				module dans notre fichier de travail.
		Les deux types d'exportation sont :

			"Named exports - Exportations nommées" : zéro ou plusieurs exportations par module
			"Default exports - Exportations par défaut" : une par module
			//Named exports
				//------ lib.js ------
				export function square(x) {
				  return x * x;
				}
				export const sum = (x) => {
				  return x + x;
				}
				//------ main.js ------
				import { square, sum } from 'lib' ;
				import * as library from 'lib' // this means import every exported module in lib and use the alias library

			//Default exports
				l'export par défaut nous donne le droit d'exporter "un seul module par défaut par fichier".
					//------ myFunc.js ------
					export default function funcName () { ... };
					//------ main.js ------
					import myFunc from 'myFunc';
		NB: "named export" est la bonne façon de procéder lorsque "nous importons une variable d'un fichier, nous devons spécifier le 				même nom mentionné dans l'autre.

		Les variables résolvent le problème du code redondant. 
		Mais cela "rend le code plus long et plus difficile à lire". "C'est là qu'intervient la déstructuration" .
